## 🚀 About Me

I've spent 15+ years across the full software lifecycle — requirements, design, development, testing, operations, maintenance, and customer-facing support — mainly on enterprise Java systems for large clients (telecom, logistics, retail infrastructure, finance). I've also led a team as a project leader.

In 2025 I made a deliberate shift toward modern technology: I earned **three AWS certifications in three consecutive months**, then moved on to hands-on practice with **TypeScript and Python** — building, testing, and publishing small projects in public, in both Japanese and English.

> 💡 **My philosophy:** Start with a small thing that *works*, test it, then add features. Solve problems at the root, not just the surface.

## 🛠️ How I Learn (and why it matters)

My reskilling isn't passive tutorial-watching — it's a deliberate, output-driven method:

- 🧑‍💻 **I write the code myself, then have AI review it** — I don't let AI write it for me.
- ✅ **I design my own tests** (pytest / Jest) and debug independently.
- 🔍 **I verify, not trust** — when an AI suggestion looks off, I confirm it with the actual commands and behavior.
- 📝 **I learn in public** — every project ships with a bilingual README + a LEARNING_LOG, and an article on Zenn (JP) / dev.to (EN).
- ⚡ **I move fast** — 10+ small projects completed in about two weeks after starting in May 2026.

## はじめまして、服部 友哉です 👋

IT業界で15年以上、要件定義・設計・開発・テスト・運用保守・顧客折衝まで、システム開発の**全工程**を経験してきました。大手企業向けのエンタープライズJavaシステム（通信・物流・小売インフラ・金融など）が中心で、プロジェクトリーダーとしてチームマネジメントも担っています。

2025年からは**モダン技術へのシフト**を本格化。AWS資格を3ヶ月連続で3つ取得し、その後 **TypeScript・Python** の実践学習へ。小さく動くものを作って・テストして・日英で公開する、というスタイルで学びを積み重ねています。

> 💡 **開発思想:** まず最小構成で動くものを作ってテストし、問題なければ機能を足していく。障害は表面的な修正ではなく、根本から解決する。

### 学習スタイル
- 🧑‍💻 **AIにコードを書かせず、自分で実装してAIにレビューさせる**
- ✅ **テスト設計まで自分で行う**（pytest / Jest）／デバッグも自己解決
- 🔍 **AIの指摘を鵜呑みにせず、コマンドで事実確認する**
- 📝 **学びを公開**（各プロジェクトに日英README＋LEARNING_LOG、Zenn・dev.toで発信）
- ⚡ **スピード感**：2026年5月開始から約2週間で10本以上のプロジェクトを完走
